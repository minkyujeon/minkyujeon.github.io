Autoresearch De Novo Protein Design

Minkyu Jeon  ·  Princeton University Collaborator: Jinyeop Song Updated: April 2026 In Progress
Project Goal

Apply an LLM + tree search (SCORE-style) framework to automatically discover optimal hyperparameter configurations for de novo protein binder design — without human-in-the-loop decisions. The system iteratively runs experiments, reads structured results, and improves its search policy across rounds.

Part I Background

Background

The Proteina-Complexa model (ICLR 2026 Oral) jointly co-designs protein structure and sequence via flow matching, guided at inference time by AF2 rewards through beam search (TTO). The model has 5 configurable TTO algorithms and a large hyperparameter space: beam_width, n_branch, nsamples, checkpoint placement, and algorithm choice (single-pass, beam-search, best-of-N).

The paper (Figure 7) shows that the optimal algorithm depends on target difficulty: Best-of-N wins on easy targets; beam search wins on hard targets. But no system automatically selects the right strategy given a new target.

The SCORE Framework

Our methodology is inspired by SCORE (Google DeepMind, 2025 — arXiv:2509.06503): an LLM + PUCT tree search system that iteratively generates and improves scientific software to maximize a quality metric. SCORE achieved SOTA on 6 scientific benchmarks including 40 novel bioinformatics methods.

SCORE ConceptOur Analog
LLM generates codeLLM generates/selects TTO config
PUCT tree search over code variantsTree search over beam configs
Quality metric (benchmark score)Unique successes / hits-per-GPU-hour
Recombination of top solutionsHybrid configs from best per-target winners
Iterative improvementMulti-round campaign with memory

Our Approach

Two complementary directions running in parallel:

  • Autoresearch (this page): SCORE-style automated sweep + analysis + LLM-guided config selection
  • LLM-Guided Adaptive TTO: Real-time routing using step-65 intermediate signals

The Autoresearch pipeline:

  Define config space (A–G beam params, notto baseline)
       ↓
  Run systematic sweep across N targets × M configs (parallel SLURM jobs)
       ↓
  Auto-generate structured analysis (JSON + figures + HTML reports)
       ↓
  LLM reads structured reports → identifies best config per target class
       ↓
  Next round: LLM proposes new hybrid configs to test

Part III Findings & Next Steps

Key Findings

Finding 1 — E_fine_selector is the most versatile config: Wins 7/10 proteins by both best single sample and mean reward. 6 fine-grained checkpoints with n_branch=3 enables frequent pruning of poor trajectories.
Finding 2 — Lookaheads beat finals in all configs: The best intermediate candidate always outperforms the mean of final survivors. The beam search pruning step is suboptimal — it discards good candidates. This is the direct gap that LLM-guided adaptive routing addresses.
Finding 3 — Target difficulty is config-dependent: BetV1 (hard, best i_PAE=0.205) struggles across all configs. IFNAR2 (easy, best i_PAE=0.130) succeeds with almost any config. The gap between easy and hard targets motivates adaptive routing.
Missing baseline: The Autoresearch sweep only ran beam search configs — no Best-of-N (no-TTO) baseline. Our companion experiment (LLM-Guided Adaptive TTO) adds this baseline (N=200) across the same targets to complete the Figure 7 comparison.

Next Steps

  • Add no-TTO N=200 baseline to complete easy/hard × algorithm comparison
  • Extract step-65 reward statistics from existing beam search logs as difficulty predictors
  • LLM routing experiment: feed step-65 JSON → LLM classifies difficulty → routes to best config
  • SCORE-style outer loop: LLM proposes hybrid configs (e.g., E_fine_selector n_branch + C_deep_exploiter beam_width) for round 2
  • Ablation: LLM routing vs. rule-based threshold (step-65 mean < -0.35 → hard → beam) vs. random
Publishable direction: Show that LLM-guided adaptive routing across the beam search config space achieves better hits-per-GPU-hour than any single fixed config — verified across the full 12 easy + 7 hard target split from the Proteina-Complexa paper.
