Pipeline Diagram

1. Slurm / CLIpython -m autoresearch_v2 init optionally researches ideas, then run starts tree search. The current full script requests 4 GPUs.
2. IdeaResearcherQwen uses local tools: target metadata, paper/reference summaries, prior run evidence, and search-history tools. It finalizes ResearchIdea objects into the store.
3. Seed Programbuild_profile_program() creates a root ConfigOnlyInferenceProgram. Current full run uses seed_profile=deep and n_candidate=16.
4. PUCT TreePuctTree selects a parent, tracks visits, scores, virtual loss, and backs up rank keys after each evaluation.
5. Node IdeasFor LLM runs, v2 can ask IdeaResearcher again at each selected parent using current feedback, compact run history, and local tree context.
6. ConfigRefinerQwen calls propose_refinement. The refiner applies schema updates, validates allowed paths, rejects duplicates, and logs LLM interactions.
7. Validation GateCurrent v2 accepts only stage_index=0. append_stage and stage_conditions are disabled because the executor still runs one stage.
8. Store ChildAccepted children are content-addressed, saved under programs/, optionally attached to idea ids, and added to the tree before worker launch.
9. GPU WorkerProgramExecutor compiles the child to Hydra overrides and launches python -m proteinfoundation.generate with one Complexa subprocess per program.
10. EvaluationThe evaluator reads Complexa result CSVs, computes successes, best iPAE/pLDDT, rank key, diversity fields, and writes score records.
11. FeedbackScores, stderr tails, proposal records, and tree summaries are persisted. The next LLM call sees failures such as OOM as prompt feedback.
12. ResultsAt budget end, v2 writes results/summary.json, best_result.json, final_candidates.json, and tree snapshots.

What Qwen Actually Controls

In v2, Qwen does not mutate model weights and does not directly write arbitrary code in the current path. It proposes structured config edits to a typed ConfigOnlyInferenceProgram. The compiler then converts that program into Hydra overrides for the frozen Complexa generator.

  • basic: beam width, branch count, sample count, and batch size.
  • elaborate: basic fields plus sampling.n_recycle.
  • full: non-templated Tier-A fields except fixed sampling.guidance_w/sampling.ag_ratio, plus sparse families such as model overrides, reward weights, and refinement loss weights.

Baseline Reuse Policy

The cleanest current-v2 comparison is to match the exact full-run CLI shape and change only the refiner:

LLM search:    --refiner llm    --budget-evals 30 --n-candidate 16 --seed-profile deep
Random search: --refiner random --budget-evals 30 --n-candidate 16 --seed-profile deep

Do not launch that random baseline until the pending full Qwen run either starts producing comparable data or finishes. The seed evaluation will be included inside the full run once it starts, and older paper_beam_K100_n96/paper_gh_1pct_n96 artifacts are not clean current-v2 CLI baselines.
