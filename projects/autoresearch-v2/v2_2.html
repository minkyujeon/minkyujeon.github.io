Autoresearch v2.2

A clean rebuild of the LLM-guided test-time optimization pipeline for de novo protein binder design. Wraps a fixed Proteina-Complexa backend with an analyst → refiner → verifier → executor → supervisor loop. Config-first: the LLM proposes JSON inference programs, never edits backend source.

What changed from v2

Scope kept

  • SCORE-style tree search with parent_portfolio (exploit / explore / rescue / backup).
  • Local Qwen3.6-27B-FP8 served via vLLM on one GPU; remaining GPUs run Proteina-Complexa children.
  • Schema-first proposals; deterministic verifier in front of every launch.

What v2.2 adds

  • Capability-manifest-as-protocol with hard / soft bound severity.
  • V8 evidence enrichment: proposal_history + operator-family fatigue warnings.
  • Primary-success-rate cap (= 2.0) to stop one hot child from dominating the portfolio.
  • Multi-route per iteration: --worker-gpu-ids 1,2,3 launches N children in parallel.
  • Analyst modes: three_experts, single_phd, three_phds, two_phds_one_system.
  • Per-iter codegen with diversification cues (sandboxed analyze() over recent runs).
  • Refiner-as-scribe with one repair attempt when JSON fails.
  • 81 unit tests (schemas, store, manifest, adapter, search policy, roles, controller, reports).

Pipeline at a glance

1 · Evidence TargetSpec + capability_manifest + last N proposals + parent_portfolio + fatigue warning + (optional) computed_analysis.
2 · Analysts Domain / ML / System experts, or 1–3 PhD-style analysts. Outputs are free-text findings + suggested operator families.
3 · Refiner Strict-JSON proposal batch. One repair attempt if schema invalid. Prompt sees budget urgency.
4 · Verifier Manifest hard / soft bounds, budget guard, manifest_bounds summary. Hard fail → blocked, soft fail → warning.
5 · Executor Local Complexa runner. --worker-gpu-ids launches N children in parallel per iteration. Timeout floor clamped ≥ 1 min.
6 · Score EvalReport → primary success rate (capped at 2.0) + iPAE + diversity. Run-store write per child.
7 · Search update Tree update + parent_portfolio rebalance + V8 fatigue detection (aggressive ≥ 2/6 or conservative ≥ 3/12).
8 · Supervisor Decides continue / pivot / stop. Decision + parsed JSON kept in transcript for audit.
Per-iteration codegen (--evidence-codegen) inserts an extra Qwen call before step 1: it writes a small analyze() function over recent run results and the sandbox executes it; the structured output joins the next EvidencePack as computed_analysis, plus a diversification cues block listing untried / stuck / successful operator families.

How to run

1 · Start vLLM (one GPU)

python -m vllm.entrypoints.openai.api_server \
  --model /path/to/Qwen3.6-27B-FP8 \
  --served-model-name Qwen/Qwen3.6-27B-FP8 \
  --host 127.0.0.1 --port 8000 --max-model-len 131072 \
  --gpu-memory-utilization 0.90 \
  --default-chat-template-kwargs '{"enable_thinking": false}' \
  --trust-remote-code

2 · Launch the pipeline (1 worker GPU, ~12 h)

PYTHONPATH=src python -m autoresearch_v2_2.pipeline run-local \
  --config configs/ar22_pdl1_search12h_v3wallclock.json \
  --run-dir runs/pdl1_demo \
  --llm-base-url http://127.0.0.1:8000/v1 \
  --llm-model Qwen/Qwen3.6-27B-FP8 \
  --worker-gpu-id 1

3 · Multi-route (4 GPU H100, 24 h)

sbatch slurm/search_4gpu_h100.slurm                # three_experts (default)
sbatch slurm/search_4gpu_h100_codegen.slurm        # + per-iter codegen
sbatch slurm/search_4gpu_h100_single_phd.slurm     # 1 PhD-style analyst

GPU 0 hosts vLLM; GPUs 1-3 are passed via --worker-gpu-ids 1,2,3 and execute children in parallel each iteration.

4 · Aggregate the run report

PYTHONPATH=src python -m autoresearch_v2_2.pipeline report \
  --run-dir runs/pdl1_demo

Design notes

Capability manifest as protocol

The backend declares mutable_fields with bound_type ∈ {hard, soft}. The verifier routes hard violations to blocking_reasons and soft ones to warnings. The Refiner sees a compact manifest_bounds summary (type, range, allowed values, risk) so it can stay inside the box without reading the full manifest.

V8 evidence enrichment

EvidencePack carries proposal_history (last N proposals + outcomes) and operator_family_fatigue_warning when an operator family has been tried repeatedly with no success. Toggle off with --baseline-evidence.

Score cap

search/scoring.py caps the primary success rate at 2.0 so one hot child cannot collapse parent_portfolio into a single line of attack.

Refiner-as-scribe

Analysts speak in free text. The Refiner converts that into the strict-JSON Proposal batch the verifier checks. If schema parse fails, one repair attempt at temperature=0; otherwise the iteration falls through to deterministic rescue.

Run store on disk

Every iteration writes iterations/iter_NNN/ with evidence_pack.json, analyst_*.json, refiner.json, verifier.json, transcript.jsonl, plus a top-level iteration_summary.json for fast aggregation.

Subgit policy

subgit/proteina_complexa is gitignored. Each developer points it at a local clone of Proteina-Complexa; the package never edits backend source during normal search.
