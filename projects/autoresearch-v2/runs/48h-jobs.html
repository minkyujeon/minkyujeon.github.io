How to read this page

  • succ / GPU-h is the cleanest cross-job comparison: total audited successes divided by total executed GPU-hours. Faster targets (PDL1) and smaller proposals naturally produce more.
  • min iPAE is the lowest interface PAE produced by any candidate the agent ran. This tells you whether the agent ever found a binder that could succeed, even if it didn't replicate.
  • Operator family bars show how many child runs the agent allocated to each search strategy. Diverse coverage is good; collapse to one family is a warning sign.
  • Cumulative audited successes sparkline shows when in the run the discoveries happened — early bursts vs. late discoveries.

Methodology notes

  • Score is a per-child compute-normalized rate: audited_unique_success / gpu_hours, capped at 2.0, plus small soft bonuses for low iPAE / high pLDDT / low duplicate fraction. Cap exists to prevent a single fast small-denominator winner from anchoring the parent portfolio.
  • "Audited" successes are the AlphaFold2-evaluated unique candidates that meet iPAE ≤ 0.18 AND pLDDT ≥ 0.90, after de-duplication on sequence. "Online" successes use a softer per-fold criterion that the backend reports during sampling.
  • PUCT is the AlphaZero-style upper-confidence-bound parent selector. Q is the per-child score; visit count is the number of expansions per parent.
  • Calibration injection reads each role's predicted-vs-actual bias on best_i_pae / n_successful / expected_gpu_minutes accumulated so far, and adds it to the analyst's user prompt at iteration ≥ 5 with a "you have over-predicted X" warning. It is removed from the Refiner and Supervisor prompts (decision-safe) so calibration data does not leak into selection.
  • Failure modes include hard failures (timeout, OOM, backend exception) and soft failures (zero_success, near_miss_interface, duplicate_collapse, proxy_mismatch). Both kinds are surfaced to analysts via proposal_history in the EvidencePack.
