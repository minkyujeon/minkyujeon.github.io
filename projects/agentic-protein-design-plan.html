Inference-Time Software Evolution for Protein Design

An LLM-driven tree search that writes and recombines inference-only protein-design programs over a frozen registry of pretrained generators, inverse folders, structure predictors, and refinement tools. The Proteina-Complexa TTO scaffold is one seed among several; the method remains useful as the underlying models evolve, because the contribution is the evolver, not any single checkpoint.

1. Objective Diagnosis of the Prior Plan

The 2026-04-15 plan correctly moved the project from a one-step routing controller to a search over inference software. But it anchored that search to a single frozen Proteina-Complexa checkpoint. Two evaluations forced a second revision.

What was right
Mapping onto the empirical-software paper: idea injection, code mutation, sandbox execution, scoring, tree search, backpropagation, and matched-budget evaluation were all in the correct place.
What was wrong (1)
A single frozen checkpoint as the experimental anchor ties the method's lifespan to that checkpoint's SOTA window. If RFDiffusion-X, Chroma-3, or Proteina v2 ships, the paper's framing collapses.
What was wrong (2)
A strict seven-function bounded API (prepare_target through should_stop) locked the search into one architectural template. The paper's biggest wins (e.g. BBKNN + ComBat recombination) came from unexpected cross-template merges.
What was wrong (3)
"Complexa-native TTO" sat on top of the motif hierarchy with BindCraft / hallucination / diffusion-trajectory control as auxiliaries. That reads as a Proteina-Complexa sequel; the user explicitly does not want that.
What stays constrained
Model weights are never trained. The entire system operates at inference only. The TTO scaffold tradition is preserved.
What is now mutable
The inference composition program itself, including which pretrained model is called, in what order, with what TTO moves, and under what selection and escalation logic.

Bottom line. Keep the TTO tradition and the matched-budget discipline. Replace "frozen single checkpoint" with "frozen registry of pretrained models". Demote the bounded API from a hard contract to a loose I/O contract. Reposition Proteina-Complexa as one seed scaffold among several.

2. Revised Thesis and Three-Tier Boundary

Thesis. Protein-design empirical software can be cast as an inference-time scorable task whose solution is a Python program that composes a frozen registry of pretrained protein tools, augmented with a library of inference-time operators. An LLM-driven tree search evolves, mutates, and recombines these programs under matched compute, using Proteina-Complexa's TTO scaffold as one of several seeds.

Core framing. Frozen tool registry and seed scaffolds, mutable inference composition. The LLM does not retrain any model. It writes programs that call pretrained models as black boxes and stitches them together with inference-time moves borrowed from across the literature.

3. Why This Is Not a Proteina-Complexa Sequel

This is a deliberate, checkable property of the design. The method must still work if Proteina-Complexa is deprecated, and it must incorporate new generators without a paper rewrite.

Design property

Model-slot-ability

The registry exposes generators, inverse folders, predictors, and refinement tools behind a common wrapper contract. Adding a new model means adding one wrapper, not changing the evolver.

Design property

Seed plurality

Five seeds start the tree. Proteina-Complexa is one of them. The top program need not contain it; if it does not, that result is itself evidence that the evolver is method-agnostic.

Design property

Cross-paradigm recombination

Recombination prompts pair programs from different method traditions (diffusion × hallucination, flow-matching × repair) rather than nudging a single lineage. Diversity is structurally enforced.

Design property

Tool-ingestion loop

When a new generator or refinement method publishes with code, a lightweight wrapper is written and the registry grows. The evolver re-runs and the living leaderboard updates without altering the paper's method.

Testable claim. Dropping any single model from the registry should degrade absolute score but not destroy the method. The evolver should still discover a competitive composition using the remaining registry. This is a required ablation, not a nice-to-have.

4. Intro and Contributions Draft

Draft Intro

Recent work shows that large language models can search over program space by proposing, executing, and revising code under an automated score. We adapt this idea to protein design by treating inference-time control as executable empirical software over a frozen registry of pretrained generators, inverse folders, structure predictors, and refinement tools. Starting from a small set of seed programs drawn from different method traditions, the system writes and recombines inference-only Python programs that compose these tools and a library of TTO moves. Each candidate is executed in a reproducible sandbox and scored under matched compute on fixed protein-design benchmarks. Tree search drives mutation, recombination, and retrieval-conditioned idea injection. The result is a living inference-time design system whose method does not depend on any single checkpoint and whose catalog can absorb new models and new inference-time tricks without retraining.

Draft Contributions
  • We formulate protein design as an inference-time scorable search over Python programs that compose a frozen registry of pretrained protein models.
  • We specify a loose I/O contract and a concrete TTO move library so that programs are portable across targets and tasks without committing to a single architectural template.
  • We demonstrate cross-paradigm recombination: pairwise hybrids of diffusion, flow-matching, hallucination, and interface-repair programs produce score jumps neither parent achieves alone.
  • We show method-agnosticity by construction: single-model and single-paradigm ablations still yield competitive programs because the evolver routes around missing tools.
  • We demonstrate a tool-ingestion loop in which adding a new pretrained model (via a small wrapper) produces a measurable, automatic score jump without changing the method or retraining anything.
  • We release seed programs, a wrapper library for current protein tools, and a living benchmark harness so that future models can be slotted in as they appear.

Paper-level consequence. The main story is LLM-guided inference-time program search over a frozen tool registry. Replay of human traces is demoted to a debugging diagnostic. Absolute state-of-the-art on any one benchmark is not the claim; the claim is that the evolver is the method, and it stays useful as the field's models and tricks evolve.

6. Tool Registry: Pretrained Models as Plug-ins

Every registry entry is a pretrained model wrapped behind a small, uniform Python interface. Wrappers are the only place where model specifics live. Adding a new model means adding one wrapper file, not touching the evolver.

Role Initial catalog Wrapper contract Extension path
Backbone generator RFDiffusion, Chroma, Proteina, Proteina-Complexa, Genie2, FoldFlow / Multiflow, FrameFlow generate(spec, n, budget) → List[Structure] with optional trajectory access for partial-diffusion moves. Any diffusion, flow-matching, or autoregressive backbone model with open inference code.
Inverse folder ProteinMPNN, LigandMPNN, ESM-IF sequence(structure, constraints, n) → List[Sequence] Any inverse-folding model that takes a backbone and returns sequences.
Structure predictor AlphaFold2, AlphaFold3, ESMFold, Boltz-1 predict(sequence, context) → Structure + confidence Any single-sequence or MSA-based predictor with reproducible scoring outputs.
Refinement / repair BindCraft, AF2-hallucination, Rosetta FastRelax, PyRosetta filters refine(structure_or_pair, diagnostics) → Structure with per-call budget reporting. Any inference-time post-generation repair or local optimization method.
Scoring / filtering AF2 pLDDT / pAE / ipAE, physics terms, FoldSeek novelty, diversity clustering score(candidate, task) → {metric: value} Any scorable inference-time signal that does not require training.

Wrapper discipline. Each wrapper declares its inference cost, memory footprint, and any stochasticity knobs. This is what makes matched-budget evaluation meaningful when the evolver composes programs that call different models.

7. TTO Move Library

These are the inference-time operators the evolver can call from any program. They are drawn from across the protein-design literature and abstracted into callables so that programs can mix them freely.

Category Operators Origin and purpose
Pre-generation set_target_context, set_symmetry_prior, set_length_distribution, set_motif_constraints Standard conditioning knobs shared across diffusion and flow-matching generators.
Trajectory control partial_diffusion_restart, branching_schedule, classifier_guidance_hook, noise_perturbation, early_stop_on_score, tempered_sampling Diffusion and flow-matching trajectory moves from RFDiffusion, Chroma, Genie2, and the broader diffusion-control literature.
Post-generation refinement bindcraft_repair, hallucination_rescue, mpnn_redesign, fastrelax_minimize, interface_optimize Local inference-time repair motifs; separate geometry acquisition from sequence and interface cleanup.
Selection and reranking af2_rerank, physics_rerank, ensemble_filter, diversity_select, novelty_filter Fixed-score proxies and diversity-aware selection; lets a program express geometry-first or physics-aware selection logic.
Meta-control escalate_budget, diversify_rescue, narrow_search, stop_on_expected_observable Orchestration operators that decide when to spend more compute, when to diversify, and when to halt.

Move discipline. Each operator has a stable signature and reports its own compute cost. Programs declare a total budget and the validator checks that the sum of move costs fits. This is what makes cross-paradigm recombination a fair search rather than a compute-spending race.

8. Seed Program Library

Five seed programs bootstrap the tree. They span different method traditions so that the initial search frontier is paradigm-diverse from step one.

Seed 1

Complexa-native TTO

generate(Complexa) with scouting, geometry diagnostics, selective refinement, and AF2 rerank. This is the closest tie to prior work and the baseline the evolver must beat.

Seed 2

RFDiffusion + MPNN + BindCraft repair

generate(RFDiffusion) → mpnn_redesign → bindcraft_repair → af2_rerank. A strong diffusion-plus-repair baseline anchored in a different tradition.

Seed 3

Chroma conditional + hallucination rescue

generate(Chroma) with class-conditional guidance, followed by hallucination_rescue on candidates whose geometry opens but sequence fails.

Seed 4

Deliberate cross-paradigm hybrid

generate(Complexa, scout) then partial_diffusion_restart with RFDiffusion on low-geometry cases, then mpnn_redesign and physics rerank. Seeds the tree with a hybrid the evolver can immediately mutate.

Seed 5

Conservative generate-and-filter

No TTO moves beyond selection. Uses the default generator and AF2 filter only. Functions as a negative control and as a compute-cheap fallback when escalation is not warranted.

Why these five. Together they cover diffusion, flow-matching-adjacent, hallucination, deliberate cross-paradigm hybrid, and a minimal baseline. This paradigm coverage at the root is what makes the PUCT exploration produce diverse recombinations early rather than drifting toward one family.

9. Search Object and I/O Contract

A tree node is an executable inference composition program. It is code, not prose. It is loosely structured: a weak I/O contract replaces the earlier strict seven-function API so the evolver can merge stages, reorder moves, or add new stages without fighting the harness. 

# Loose I/O contract - the only thing the validator enforces.
# Any internal structure is fair game.

def run_design_program(
    task: TaskSpec,                 # target, constraints, success metric
    budget: Budget,                 # wall-clock or GPU-hour quota
    registry: ToolRegistry,         # frozen pretrained models (wrappers)
    moves: MoveLibrary,             # inference-time operators
    diagnostics_cb: Callable,       # structured logging for sidecar
) -> DesignResult:
    """
    Must:
      - call only registry tools and library moves (import allow-list)
      - respect the budget (validator rejects overruns)
      - emit structured diagnostics (geometry, sequence, physics, redesign-gap)
      - return candidates with reproducible seeds

    Free to:
      - use any control flow, combine any tools and moves
      - skip, reorder, or merge canonical stages
      - branch, escalate, or early-stop on its own criteria
    """
    ...
    return DesignResult(candidates=..., diagnostics=..., budget_used=...)

Why loose. The empirical-software paper's biggest jumps came from unexpected structural merges (e.g. BBKNN with ComBat-corrected PCA). A strict per-stage API would have blocked that. Here the validator only checks safety and budget, not shape.

Operator What it does Example

11. Cross-Paradigm Recombination

The empirical-software paper reports that 44 percent of pairwise recombinations beat both parents on scRNA-seq batch integration. Our setting is harder because parents come from different method paradigms, not just different hyperparameter regions. That increases diversity and the chance of synergistic hybrids.

Parent A Parent B Expected synergy
Complexa-native TTO (scout-diagnose-escalate) BindCraft interface repair Complexa produces diverse geometry candidates; BindCraft cleans up interface before AF2 rerank.
RFDiffusion trajectory branching Hallucination sequence rescue Branching finds multiple geometry arms; hallucination recovers sequence on arms where MPNN produces low-confidence designs.
Chroma class-conditional guidance Physics rerank + ensemble filter Class guidance narrows the generator's output to target-relevant families; physics rerank suppresses failure modes class guidance misses.
Diffusion partial-restart control Diversity-aware selection Restarts rescue low-geometry cases; diversity selection prevents collapse to the first lucky arm.

Method-agnosticity as a signal. The most interesting outcome is a top program whose best ablation removes Proteina-Complexa and still outperforms every seed. That is exactly the story the paper should tell.

12. Evaluation and Method-Agnosticity Tests

The main result is searched-program quality under fixed compute, plus structured evidence that the method does not depend on any single registry member.

Mode Question answered Status in this plan
Live matched-budget search Does the searched program frontier beat strong static baselines at equal compute? Primary scientific benchmark.
Registry ablations Does the evolver still produce competitive programs when one model or one paradigm is removed from the registry? Primary evidence for method-agnosticity.
Tool-ingestion experiment Does adding a new model at run N yield a measurable score breakthrough by run N + k? Primary evidence for the living-leaderboard claim.
Cross-paradigm recombination rate What fraction of cross-paradigm hybrids beat both parents? How does that compare within-paradigm? Secondary, directly tests the recombination story.
Replay Can the system mutate a historical program to a better one on the same trace? Debugging diagnostic only. Not a headline claim.

Primary metrics

  • Unique successes at matched budget on each scorable task.
  • Success per GPU-hour as an efficiency view of the same frontier.
  • Cross-paradigm recombination win rate: fraction of hybrids that beat both parents.
  • Registry-ablation robustness: score degradation when any single model is removed.
  • Tool-ingestion latency: iterations between adding a new model and a measurable score jump.
  • Diversity among successes to confirm programs are not collapsing to one family.

Baselines and ablations

  • Each seed program run standalone at matched budget.
  • Human-written hybrid programs without LLM mutation.
  • Config-only search over a fixed program template (no code-level mutation).
  • No-recombination ablation.
  • No-literature / no-archive injection ablation.
  • Registry ablations: drop one model, drop one paradigm, drop all refinement tools.
  • Tool-ingestion time series: registry at month 0, +1, +2, ... as new models are added.

13. Implementation Plan

Phase 0

Freeze task, metric, budget

Pick the first scorable task (binder design is the primary candidate), finalize success metric and matched-budget unit, and define the import allow-list.

Phase 1

Build the tool registry

Write wrappers for the initial catalog: backbone generators (Complexa, RFDiffusion, Chroma), MPNN variants, AF2 / ESMFold, BindCraft, and core filters. Record inference cost per call.

Phase 2

Write the TTO move library

Implement the operators listed in section 7 with stable signatures. Include a compute-cost self-report for each so budget accounting works.

Phase 3

Author five seed programs

Code the five seeds in section 8 against the loose I/O contract. Run each standalone to confirm they produce valid designs at matched budget and to collect baseline numbers.

Phase 4

Mutation harness

LLM prompt with program, diagnostics, and injected ideas; compile; execute; score; archive. Reproduces the paper's inner loop, adapted for protein inference.

Phase 5

PUCT tree + recombination controller

Add rank-normalized PUCT across program nodes and the cross-paradigm recombination prompt. Track visits, rewards, and sidecar metadata.

Phase 6

Ablations and ingestion experiment

Registry-drop ablations, cross-paradigm recombination win-rate analysis, and a tool-ingestion time series adding at least one new model mid-run.

Phase 7

Second scorable task

Motif scaffolding or symmetric oligomer design, to demonstrate portability of the evolver across tasks with no paper-level change.

14. Risks and Open Questions

Risk Why it could hurt the paper Mitigation in this plan
Compute blow-up Programs that stack expensive models can dominate the search. Every wrapper and move reports cost; validator rejects budget overruns; matched-budget scoring penalizes compute-heavy programs by construction.
Reward hacking AF2 confidence and related proxies can be inflated without real binder success. Diagnostic sidecar preserves geometry, physics, and diversity signals; manual inspection of top programs before reporting.
Paradigm collapse Tree search may converge to one generator and one refinement idiom. Cross-paradigm recombination controller, paradigm-diverse seeds, and diversity-aware selection operators.
Wrapper drift New models may have inconsistent APIs that make portability illusory. Wrapper contract declares cost and stochasticity; a small test suite runs on every new wrapper before registry admission.
Ingestion overfitting Adding a new model mid-run could bias scoring toward that model by sampling opportunity. Report pre- and post-ingestion score trajectories separately; include a held-out target set for the ingestion experiment.
Framing drift back to Complexa Writing and narration pressure could re-anchor the paper to a single checkpoint. Registry-ablation and tool-ingestion results must appear in the main paper, not an appendix.

Hard constraint. Nothing in this project retrains any model. If a proposed operator or experiment requires weight updates, it is out of scope and must be either reworked as an inference-time move or deferred to a future paper.
