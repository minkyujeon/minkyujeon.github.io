1. Read This First

Core decision: the right object is not a generic protein-research agent and not a binder generator. The right object is an agentic TTO controller around a fixed Proteina-Complexa model.

What is fixed
Proteina-Complexa checkpoint, evaluation criteria, SLURM environment, and the real search / refinement operators that already exist in the local stack.
What the agent changes
Which TTO operator to run next, how to use budget, when to escalate, and how target-specific context should influence that decision.
What literature does
It enriches target context. In replay it should not override same-target local trace evidence. In cold-start it becomes one of the main target-specific evidence sources.
What success means
Better next-step decisions under matched budget, then eventually better hits-per-GPU-hour or matched-budget unique successes than static or rule-based baselines.
Current clarification
Historical replay and cold-start target understanding are now treated as two different benchmarks. The replay result is useful, but it is not evidence of round-0 target understanding.

Concrete evaluation of the current plan

Strong

Correct unit of decision

The agent is judged on state -> diagnosis -> next action -> outcome, not on vague biological commentary and not on one-shot binder generation.

Strong

Deterministic boundaries

Context enrichment, state building, compilation, and validation are deterministic. The LLM owns only the diagnosis and next-action proposal stage.

Strong

Executable action space

The controller proposes only real actions already tied to the local stack: beam variants, MCTS, sequence hallucination, reranking, or stop/collect-more-data.

Needs discipline

Literature as enrichment, not master signal

The plan stays correct only if target-specific literature remains a context layer below same-target local evidence. Otherwise the controller will overfit to generic biology.

Needs discipline

Local-first retrieval stack

Free web retrieval should start with structured Europe PMC queries and local note caches, not unrestricted browsing. The goal is auditable retrieval, not chatty search.

Now concrete

Contracts exist

The project now has concrete schemas and scripts for TargetContextBundle, TTOState, TTOActionProposal, and TTO Run Manifest.

Bottom line. The plan is now rigorous enough to support both a real offline replay controller and a stricter cold-start benchmark. The main remaining work is not architecture; it is keeping those two evaluation modes separate and comparing each honestly against the right baseline.

2. Thesis and System Boundary

The central thesis is narrow and testable: an LLM can improve inference-time scaling in Proteina-Complexa by choosing better test-time actions. The model does not need to generate protein binders directly. It needs to read a compact, tool-grounded state and propose the next executable TTO move under a fixed budget.

Surface Fixed Variable / agent-controlled
Model layer Proteina-Complexa checkpoint, evaluation script, success thresholds None in v1 of the controller
TTO layer Real search families already in the stack Which operator to run, whether to escalate, when to stop
Context layer Known hotspots, local structures, prior run artifacts Whether to enrich with literature snippets, prior episode retrieval, and structure/sequence analysis
Budget layer Total GPU budget and hardware availability Budget split across initial arms or later rounds

Why the nanobody paper still matters

The agent-guided nanobody work is relevant for one reason only: it uses the LLM as an orchestration layer over deterministic evidence. That division of labor transfers well. Its domain-specific wet-lab funnel, hotspot discovery logic, and candidate-selection details do not need to be copied literally.

3. Pipeline Diagram

Fixed Assets

Proteina-Complexa checkpoint, success metric, evaluation script, per-target compute budget, and SLURM execution environment.

↓

Stage A0: Context Enrichment

Known hotspots, target structures, free literature retrieval, local notes, sequence analysis, and structure analysis produce a TargetContextBundle.

Europe PMC Biopython-style parsing Local notes
↓

Stage A: State Builder

Prior run CSVs, diagnostic sidecars, attempted operators, and budget metadata are compressed into a compact TTOState.

→

Stage B: Gemma 4 31B

The LLM reads the state, forms a bottleneck hypothesis, and emits a single structured TTOActionProposal with evidence, confidence, and fallback.

↓

Stage C: Deterministic Compiler + Validator

The proposed action is accepted only if it matches the operator registry, target ID, round index, and positive remaining budget. Output: exact Hydra overrides and TTO Run Manifest.

↓

Stage D: Run, Summarize, Reflect

Proteina-Complexa runs, evaluation outputs are summarized, and the next trace episode is written for replay or live control.

→

Episode Store / Memory

Each round stores state, diagnosis, action, outcome, and reflection. This supports offline replay first and live control later.

↓

Comparison Baselines

Static default policy and existing rule-based routing policy stay in the loop as the real baselines to beat.

The crucial architectural choice is that only the diagnosis / action proposal stage is LLM-owned. Everything else is deterministic, auditable, and replayable.

4. Executable Contracts

The plan is only real if every stage emits a compact artifact that the next stage can consume without human interpretation. The current working execution spec is centered on four required objects.

Artifact What it contains Why it matters
TargetLiteratureRetrieval Structured results from free Europe PMC queries: title, abstract, year, citation count, DOI/PMID/PMCID, and query provenance. Gives target-specific prior context without forcing raw browsing into the LLM prompt.
TargetContextBundle Hotspot-aware structure summaries, chain counts, chain lengths, compact local notes, and optional retrieved literature summaries. Bridges biology / structure information into the routing stack.
TTOState Target context summary + diagnostic summary + attempted operators + budget state + provenance. Defines the exact LLM input for a routing decision.
TTOActionProposal Diagnosis label, bottleneck hypothesis, recommended next action, fallback, expected observable, confidence, and evidence. Forces the LLM to make one falsifiable, executable recommendation.
TTO Run Manifest Exact Hydra overrides, run name, run directory, nsamples, and compile-time validation flags. Eliminates ambiguous interpretation of LLM prose before launch. 
TargetLiteratureRetrieval
  -> TargetContextBundle
  -> TTOState
  -> Gemma 4 31B
  -> TTOActionProposal
  -> TTO Run Manifest
  -> Proteina-Complexa run
  -> TraceEpisode

Hard rule. The compile step must fail closed. Invalid actions are rejected. They are never silently mapped to a nearby action.

5. Free Retrieval and Analysis Stack

Literature retrieval is useful only if it is structured, multi-source, and compressed into decision-facing fields. The goal is not to dump papers into the prompt. The goal is to build a compact target-context object that helps the model form a first TTO hypothesis.

Free API

Multi-source paper search

The next retrieval layer should aggregate PubMed, PMC / Europe PMC, bioRxiv / medRxiv, arXiv, and optional local PDF caches, then deduplicate and rerank before anything reaches the model.

Local analysis

Structure and sequence parsing

Target structures should be parsed into chain lengths, hotspot coverage, residue neighborhoods, surface / loop proxies, multichain flags, and structural risk flags before they reach the model. The goal is evidence compression, not raw coordinate dumping.

Control principle

Field contract comes first

Before expanding retrieval breadth, the project should fix exactly which identity, structure, hotspot, and literature-claim fields are allowed into the LLM context.

Recommended multi-source tool order

Tool Role Output Why it belongs
retrieve_target_literature.py Target-conditioned literature retrieval TargetLiteratureRetrieval Should expand from Europe PMC-only to PubMed / PMC / bioRxiv / arXiv aggregation with dedup and rerank
build_target_context_bundle.py Context enrichment TargetContextBundle Merges target identity, hotspots, structures, sequence features, risk flags, and literature claims
build_diagnostic_sidecar.py Run artifact parsing Target diagnostic summary Used only after at least one same-target round exists; not allowed in cold-start mode
build_tto_state.py State compilation TTOState Defines the exact LLM input contract

What should actually go into the LLM context

Field group Examples Why it matters
Target identity display_name, aliases, gene/protein synonyms, chain IDs, total residues Necessary for query expansion, deduplication, and exact-match literature reranking.
Hotspot context known_hotspots, hotspot residue types, local neighbor density, surface / loop proxy Helps the agent reason about accessibility, geometry, and whether simple beam search is likely to be enough.
Structure summary chain lengths, multichain flag, hotspot coverage, residue composition, optional secondary-structure context Provides the minimum structural picture without dumping raw coordinates.
Structural risk flags conformation-sensitive, long flexible loop, glycan/PTM warning, interface accessibility warning These are the strongest cold-start hints for whether the first action should be conservative or exploration-heavy.
Literature retrieval summary source, PMID/DOI/arXiv id, title, year, query, exact-match flag, why-kept Keeps provenance auditable and prevents the model from treating all papers as equally relevant.
Literature claims interface/epitope claims, conformation/state claims, known binder precedent, negative constraints The model should reason over claims, not over raw abstract text.
Decision-facing summary compact_facts, warnings, structural hypotheses, cross-target priors if allowed This is the final compressed context that should directly support the first TTO hypothesis.

Cold-start retrieval sources

Do not start with unrestricted browsing or raw abstract dumps. The retrieval layer should normalize, deduplicate, rerank, and compress evidence into explicit fields and claims before the model sees it.

6. Target-Side Hardness Priors

When hard targets show low interface metrics such as iPAE and pLDDT, the useful cold-start question is not simply “is this target hard?” The better question is: what target-side properties make interface formation likely to be unstable, ambiguous, or state-dependent?

Interpretation rule. These priors are not meant to replace actual same-target runs. They are meant to provide a rational prior for the first TTO action before any same-target evidence exists.

Proxy groups worth extracting

Proxy group Example fields Why it matters for hard targets
Flexibility / state uncertainty missing residues near hotspot, B-factor summary, altloc fraction, loop fraction, multi-structure hotspot RMSD, AFDB hotspot pLDDT, optional BioEmu hotspot RMSF High target flexibility can make the hotspot poorly defined, so low interface metrics may reflect target heterogeneity rather than only poor search.
Hotspot accessibility difficulty hotspot patch compactness, surface exposure proxy, recessedness / depth proxy, fragmentation, distance span, neighbor density A diffuse, fragmented, or partially buried hotspot is harder to hit with a compact de novo binder and may justify broader search at round 0.
Biochemical occlusion glycosylation near hotspot, PTM warning, disulfide-rich neighborhood, multimer overlap, receptor overlap, membrane proximity The hotspot may be nominally known but practically difficult to access because the local biochemical context blocks or constrains binding.
Interface geometry burden multichain requirement, discontinuous hotspot segments, local curvature proxy, required contact span, charge heterogeneity, hydrophobic patch fraction Targets that require broad or discontinuous contact geometry are more likely to need exploration-heavy first actions.

Cheap defaults vs advanced optional tools

Tier Tools Role
Cheap / default PDB/mmCIF parsing, DSSP-style secondary structure, solvent accessibility, residue-neighborhood counts, local curvature / compactness proxies, literature claim extraction Should be the default cold-start layer because these are inexpensive and deterministic.
Advanced / optional AFDB lookup, multiple experimental structures, BioEmu target ensembles, MD-derived mobility summaries, curated PTM / glycan annotation Useful when state uncertainty is the main suspected difficulty, but should remain additive rather than mandatory.

How these should reach the LLM

The controller should not receive dozens of raw proxy values. The builder should collapse them into compact risk summaries such as: flexibility_risk, hotspot_accessibility_risk, state_uncertainty_risk, and geometry_burden_risk, plus a short list of top structural reasons.

Important nuance. These four target-side groups are not the final goal. They are the current best human-designed scaffold for cold-start target understanding. The long-term goal is for the agent itself to decide which of these questions matter for a given target and which tool outputs it needs next.

7. Ultimate Agent Goal

The ideal system is not one where the human permanently hard-codes a checklist such as flexibility_risk or hotspot_accessibility_risk and the LLM passively consumes it. The ideal system is more agentic: the model should infer what information is missing, decide which tool to call, read the result, and update its hypothesis.

Current practical mode

Human-designed scaffold

The current plan uses structured target-side fields because they are auditable, cheap, and reproducible. They give the controller a disciplined starting point for cold-start reasoning.

Ultimate mode

Tool-using information seeker

The long-term controller should look at a target, recognize uncertainty, ask what information would reduce that uncertainty, and then choose retrieval or analysis tools accordingly.

What the mature loop should do

Step Agent question Expected behavior
1. Initial read What about this target is currently unknown? Identify gaps such as flexibility, hotspot exposure, multistate ambiguity, or literature uncertainty.
2. Tool choice Which evidence would reduce the most uncertainty? Call retrieval, structure-analysis, alias expansion, AFDB lookup, or optional ensemble tools as needed.
3. Hypothesis update Given the new evidence, what seems to be the main target-side burden? Update the target hypothesis rather than restating generic biology.
4. First action Under the current budget, what first TTO action is now most justified? Emit one executable TTOActionProposal plus the expected observable.

Why the current structured fields still matter

Even in that more agentic future, a constrained tool registry and structured outputs are still necessary. The model should be free to decide what it needs to know, but not free to emit unauditable or non-executable actions. So the right end state is: agent decides what to ask for, while tools and output contracts remain constrained.

8. Executable TTO Action Space

The action space must stay aligned with what can actually be launched. If the current stack cannot execute an action and log it cleanly, that action should not be part of the required controller benchmark.

Action When it is appropriate Why the compiler needs it
collect-more-data Too few candidates or too much uncertainty to route confidently Lets the agent admit uncertainty without inventing a stronger action
scale-default Easy or saturated targets where escalation is more likely to add variance than gain Captures the "do not overreact" branch for easy targets like PDL1
stochastic-beam-search Geometry-limited targets that need more opening in search Provides a concrete non-generic "be more diverse" move
diverse-beam-search Ambiguous targets where preserving diversity is safer than heavier escalation Matches the current routing vocabulary and diversity-aware search logic
mcts Zero-hit frontier targets or cases where beam-family actions remain closed Represents a clear escalation step under the current stack
sequence_hallucination Sequence-limited targets where geometry is present but local sequence edits remain weak Separates geometry-opening from sequence-fixing
physics-sidecar-rerank Optional later action if physics-sidecar signals are wired and trustworthy Kept optional until the execution path is fully audited

Important constraint. The agent must emit one action from the registry, not prose such as “try something stronger” or “search more broadly.”

9. Historical Replay Result

Important boundary. The following result is a historical replay / follow-up decision benchmark. It is not a cold-start benchmark. The agent was given same-target local summaries and seeded diagnostics derived from completed human-designed runs.

Before moving to cold-start target understanding, the pipeline was tested in replay mode: feed Gemma 4 31B a compact TTOState plus a TargetContextBundle, both built from existing target summaries, and ask for one executable TTOActionProposal. This benchmark answers a narrow question: does the LLM follow the same broad routing logic that the human-designed campaign already discovered?

What the replay input actually contained

Artifact What went in Interpretation
TargetContextBundle Structure summary, hotspots, and a short human-authored same-target note summarizing the local lesson. Useful for replay, but not acceptable for round-0 cold-start because it leaks the answer.
TTOState Budget, round index, attempted operators, and a seeded diagnostic_state with self success, MPNN success, geometry pass, and dominant failure mode. This is explicitly same-target local evidence derived from completed runs.
Prompt Allowed action registry, evidence hierarchy, JSON output contract, and one-action-only instruction. Constrained the model to produce a compiler-ready next step.

What Gemma 4 31B proposed

Target Replay input signal Gemma proposal Human-designed conclusion Alignment
PDL1 self_success_rate = 1.0, dominant_failure_mode = saturated_or_default_favored, notes that high noise and hallucination hurt scale-default Do not over-escalate an already open easy target High The model correctly stayed conservative and rejected noise, hallucination, and heavier search.
CD45 Low self success, high binder RMSD pass, poor interface geometry, geometry-limited label, note that noise helped stochastic-beam-search
fallback: diverse-beam-search		 Open geometry before trying sequence-focused refinement High The model identified a geometry bottleneck and chose a search-opening action rather than hallucination.
SC2RBD Zero-hit state, geometry-limited-zero-hit label, note that beam-family runs remained closed mcts
fallback: diverse-beam-search		 Escalate earlier to heavier search for a frontier target High The model matched the intended escalation logic and did not waste another default-style round.

Replay takeaway

In replay mode, Gemma 4 31B aligned well with the existing human-designed routing logic on all three targets: PDL1 -> stay conservative, CD45 -> open geometry, SC2RBD -> escalate to MCTS. That is encouraging, but it should be interpreted narrowly. It shows that the model can read a compact same-target state and reproduce the broad conclusion. It does not yet show that the model can infer those strategies from the target alone.

10. Evaluation Ladder

Phase 0

Contract hardening

Freeze schemas, operator registry, and compile-time validation before any serious controller experiment.

Phase 1

Target context enrichment

Build real TargetContextBundle artifacts for PDL1, CD45, and SC2RBD without leaking same-target local outcomes into the notes.

Phase 2

Cold-start benchmark

Give Gemma 4 31B only target-derived context plus budget and ask for the first TTO action. No same-target local traces are allowed.

Phase 3

Offline replay benchmark

Ask Gemma 4 31B to choose the next action for completed episodes under matched budget, then score action regret and routing accuracy.

Phase 4

First-campaign planning

Allow small round-0 budget splits, but only after cold-start first-action quality is working.

Phase 5

Live two-round controller

Run one real state-action-outcome loop twice and compare against static default and current rule-based routing.

Phase 6

Retrieval ablations

Compare no enrichment vs structure-only vs structure+literature vs structure+literature+prior episodes, separately for cold-start and replay.

Primary metrics

  • Scientific: hits per GPU hour or matched-budget unique successes.
  • Agentic: cold-start first-action quality, offline routing accuracy, action regret under matched budget, and confidence calibration.
  • Ablation: gain from context enrichment and retrieval over local-trace-only routing.

12. Cold-Start Next Step

The most rigorous next move is now cold-start target understanding. Build round-0 inputs that contain only target-derived context and budget, then ask Gemma 4 31B for the first TTO action without any same-target local result summaries.

Cold-Start Inputs

Target structure, known hotspots, deterministic structure/sequence analysis, free literature retrieval, and fixed budget.

↓

Forbidden at Round 0

No same-target sweep summaries, no same-target diagnostic sidecars, no human-authored notes such as “default-favored” or “noise helps.”

→

Gemma 4 31B Cold-Start Planner

The model receives a ColdStartTTOState and must choose one first executable action with a falsifiable hypothesis.

↓

Compiler and Validator

The action is compiled only if it matches the executable registry and budget rules.

→

First Real TTO Run

The proposal becomes the first same-target evidence. Only after this step does replay/follow-up mode begin to exist.

Cold-start and replay answer different questions. The page now treats them as separate benchmarks.

Target Role in the benchmark Expected routing behavior Why it is useful
PDL1 Structured target with abundant literature See whether the model infers a conservative first move without being told it is easy Tests whether the agent can resist inventing unnecessary complexity on a target that looks tractable
CD45 Likely geometry-sensitive hard target See whether the model prefers search-opening actions over premature sequence refinement Tests whether structural context alone can suggest a geometry bottleneck hypothesis
SC2RBD Frontier-like target with rich external biology context See whether the model proposes an earlier aggressive search action under uncertainty Tests whether literature and structural context can justify stronger first-round escalation

Cold-start command skeleton

# 1. Free retrieval
python script_utils/retrieve_target_literature.py ...

# 2. Target context enrichment
python script_utils/build_target_context_bundle.py ...

# 3. Cold-start state without same-target results
#    build ColdStartTTOState from context + budget only

# 4. Gemma 4 31B prompt -> first TTOActionProposal
# 5. Compile and validate
python script_utils/compile_tto_action.py ...

# 6. First real run

Methodological rule. Do not call a result “target understanding” if the input already contains same-target human-designed conclusions. Replay remains useful, but it is a different benchmark.
