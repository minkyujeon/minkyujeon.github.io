1. The Problem: Inference-Time Scaling Plateaus

De novo protein design pipelines like Proteina-Complexa (ICLR 2026 Oral) jointly co-design protein structure and sequence via flow matching. At inference, test-time optimization (TTO) guides the denoising trajectory using AF2 rewards, steering toward binding-competent designs.

The Figure 7 pattern from the Proteina-Complexa paper reveals a critical gap:

Easy Targets

Best-of-N (random sampling) outperforms beam search. Both keep climbing with more compute. The model already knows how to bind these targets.

Hard Targets

Best-of-N plateaus at ~15 GPU hours. Beam search keeps scaling. But no one knows which targets are hard before spending the compute.

Goal: Build a system where inference-time scaling doesn't plateau — analogous to how o1/o3 achieve better results with more test-time compute in LLMs. The agent reads structured intermediate signals and adaptively allocates compute across a rich action space.

Current approaches use fixed hyperparameter strategies applied uniformly to all targets. No existing system detects easy vs. hard early and adapts. The action space is much richer than just "beam search vs. Best-of-N" — but the full space has never been systematically explored.

2. The Pipeline & Action Space

How the Pipeline Works

Proteina-Complexa generates proteins through a 4-stage pipeline: Generate (denoise from noise to structure+sequence) → Filter (rank by reward) → Evaluate (AF2 refold + ProteinMPNN redesign) → Analyze (success thresholds + diversity clustering).

Generate: t=0 (noise) ──denoise──▶ t=400 (structure + sequence) │ Beam Search: ├─ step 0: branch x n_branch │ rollout to completion │ AF2 score ──▶ keep top beam_width ├─ step 100: branch ──▶ score ──▶ prune ├─ step 250: branch ──▶ score ──▶ prune └─ step 400: final PDBs with co-designed sequences │ Filter: top-N by reward, deduplicate sequences │ Evaluate: AF2 refold (self seq) + ProteinMPNN redesign + refold │ Analyze: iPAE ≤ 7.0 AND pLDDT ≥ 0.9 AND scRMSD < 1.5A ──▶ "success"

Key insight: Only beam search and single-pass have been tested. FK-steering (soft selection via temperature), MCTS (UCB tree search), reward composition, denoising temperature, and sequence hallucination refinement are all implemented in the codebase but completely unexplored.

FK-Steering: The Continuous Spectrum

FK-steering is identical to beam search except for selection: instead of hard top-k, it samples with P(beam) ~ exp(reward / temperature).

  • T → 0: Converges to beam search (greedy exploitation)
  • T → ∞: Converges to Best-of-N (pure exploration)
  • Intermediate T: Soft exploration-exploitation tradeoff

This directly addresses the Figure 7 problem: the agent can set temperature adaptively rather than choosing between two discrete algorithms.

5. Key Findings

1. No universal winner across targets

Wide beam wins on PD1, PDL1, TrkA, CrSAS6, DerF7. Deep beam wins on CD45, Claudin1, HER2_AAV, BetV1. The Autoresearch sweep confirms: optimal config changes for 4/10 proteins when switching from top-1 to top-16 selection. Config sensitivity (CV of iPAE) ranges from 0.027 (IFNAR2, config doesn't matter) to 0.648 (BetV1, config is critical).

2. Beam search concentrates reward 3-5x vs. random sampling

Mean iPAE drops from 0.5-0.8 (notto) to 0.15-0.30 (beam search) across all targets. However, notto sometimes finds comparable best samples through sheer N=200 volume, especially for easy targets.

3. Self sequences are competitive with ProteinMPNN

Proteina's co-designed sequences pass at equal or better rate than MPNN redesigns across all evaluated targets. Self pass rate exceeds MPNN in 8/8 completed comparisons (considering best config per target). This validates that joint structure+sequence co-design works.

4. Hard targets show diminishing returns

BetV1 (best iPAE 0.220) and HER2_AAV (0.219) remain difficult even with beam search. BetV1 wide produced only 2 samples (likely timeout). CD45 achieves only 25% self pass rate with deep config. These targets need qualitatively different strategies, not just more beam search.

5. The action space is dramatically underexplored

FK-steering, MCTS, denoising temperature, reward composition, and sequence hallucination refinement are all implemented but untested. The current "wide vs. deep" comparison covers <5% of the available parameter space.

6. Experimental Plan

Experiments are ordered by expected impact and feasibility. Each experiment tests a specific hypothesis about the action space.

Phase 1: Expand the Action Space

Exp 1 FK-Steering Temperature Sweep

Hypothesis: FK-steering with intermediate temperature provides better scaling than either beam search (T=0) or Best-of-N (T=inf).

Setup: 3 targets (IFNAR2/easy, CD45/hard, Claudin1/medium) x 5 temperatures (T=0.01, 0.1, 0.5, 1.0, 5.0), beam_width=8, n_branch=4, deep checkpoints. 15 jobs.

Metric: Unique successes per config. Does intermediate T outperform both extremes?

Cost: ~15 GPU-hours

Exp 2 Denoising Temperature Sweep

Hypothesis: sc_scale_noise controls trajectory diversity independently from the search algorithm. Higher temperature + beam search may produce more structurally diverse hits.

Setup: 2 targets (PDL1, CD45) x 4 noise scales (0.05, 0.1, 0.2, 0.4) x beam-search wide config. 8 jobs.

Metric: FoldSeek cluster count among successes. Does higher noise produce more diverse hits?

Cost: ~8 GPU-hours

Exp 3 Composite Reward Signals

Hypothesis: Adding fast reward signals (TMOL H-bonds, bioinformatics shape complementarity) as secondary objectives improves pruning quality.

Setup: 2 targets (CD45, BetV1) x 3 reward compositions: (a) iPAE only (baseline), (b) iPAE + shape_complementarity (weight 0.3), (c) iPAE + H-bond count (weight 0.2) + pLDDT (weight 0.1). 6 jobs.

Metric: Pass rate improvement on hard targets. Do multi-signal rewards rescue hard targets?

Cost: ~6 GPU-hours

Exp 4 Sequence Hallucination Refinement

Hypothesis: Post-generation AF2-guided sequence optimization improves MPNN pass rate, especially where self_pass >> mpnn_pass (backbone is good but redesign fails).

Setup: Take top-16 designs from PDL1-deep (100% self, 37.5% mpnn) and CD45-deep (25% self, 15.6% mpnn). Apply sequence hallucination with default loss weights.

Metric: MPNN pass rate before vs. after refinement.

Cost: ~2 GPU-hours

Phase 2: Build the Agent

Exp 5 Diagnostic Round: Single-Pass Signal Extraction

Hypothesis: A cheap single-pass diagnostic (N=32, ~5 min) contains enough signal to predict target difficulty and route to the optimal algorithm/config.

Setup: For all 11 targets, extract from the N=200 notto runs: mean iPAE, iPAE variance, % below 0.23, reward distribution shape. Correlate with optimal config from beam search results.

Metric: Can we predict which config wins from notto statistics alone?

Cost: Analysis only (data already collected)

Exp 6 LLM Routing Validation

Hypothesis: An LLM reading structured JSON (iPAE distribution, pLDDT stats, sequence diversity, reward trajectory) can correctly route targets to configs with higher accuracy than simple threshold rules.

Setup: Feed all 11 targets' diagnostic data to Claude as structured JSON. Compare LLM routing decisions against oracle (actual best config from sweep).

Metric: Routing accuracy. Does LLM beat the rule: "if mean_iPAE > 0.5, use deep; else use wide"?

Cost: API calls only

Exp 7 Multi-Round Adaptive Pipeline

Hypothesis: An agent that reads Round 1 results and adjusts Round 2 parameters produces more unique successes per GPU-hour than any fixed strategy.

Setup: 3 targets (easy/medium/hard). Round 1: diagnostic N=32. Round 2: agent-selected config based on Round 1. Round 3: agent adjusts based on Round 2. Compare cumulative unique successes against fixed-config baselines.

Metric: Hits-per-GPU-hour scaling curve. Does the agent's curve keep climbing where fixed strategies plateau?

Cost: ~30 GPU-hours

Phase 3: Scaling Validation

Exp 8 Full Scaling Curve Comparison

Hypothesis: The adaptive agent achieves inference-time scaling that doesn't plateau, even on hard targets.

Setup: 3 targets x 5 compute budgets (5, 10, 20, 40, 80 GPU-hours) x 3 strategies: (a) fixed Best-of-N, (b) fixed beam search, (c) adaptive agent. Plot unique successes vs. GPU-hours.

Metric: Scaling exponent. Does the agent's curve have a higher slope at high compute budgets?
